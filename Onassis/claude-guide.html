4機能の一言まとめ

まず「何ができる機能なのか」を1行で把握する。詳細は各機能の解説セクションで確認しよう。

📁

Project

資料と指示を覚える専用作業部屋

同じ資料・前提で繰り返し会話するための機能。ナレッジ（参照ファイル）とカスタム指示を一度設定すれば、毎回の説明が不要になる。

誰向け：全ユーザー（非エンジニア含む）

🧩

Artifacts

会話から生まれる共有できる成果物

チャットで依頼した文書・コード・図・WebアプリをチャットサイドのウィンドウにArtifactとして生成。会話で反復編集し、URLで公開・共有できる。

誰向け：全ユーザー（コード不要でアプリも作れる）

🛠

Cowork

PC作業を任せるデスクトップ代理人

ローカルファイルの整理・変換、Excel/PPT作成、複数ステップ作業の自動実行など「作業」を丸ごと委任する機能。デスクトップアプリ必須。

誰向け：非エンジニアのナレッジワーカー中心

⌨

Claude Code

コードを書き直す開発エージェント

ターミナル上でリポジトリを直接読み書きする開発エージェント。機能実装・バグ修正・テスト作成・Git操作まで自律的に実行する。

誰向け：エンジニア・開発者（ターミナル操作が前提）

FEATURE 1

📁 Project — 資料と指示を覚えさせる

毎回同じ資料を添付・説明し直す手間をなくす機能。一度設定すれば以後のチャット品質を自動で底上げする。

できること

  • PDFやテキストファイルをナレッジとしてアップロードし、常時参照させる
  • カスタム指示でプロジェクト内の全チャットに口調・役割・出力形式を固定
  • チャットをプロジェクト単位で整理・管理
  • Team/Enterpriseプランでは部署全員とプロジェクトを共有
  • 有料プランではナレッジが上限に近づくとRAG（必要箇所だけ参照）が自動有効化され、容量が最大約10倍に拡張

作成方法（3ステップ）

  1. サイドバー「プロジェクト」→「新規プロジェクト」をクリック
  2. 名前を付けて作成（例：「2026年度 採用業務」）
  3. ナレッジにファイルをアップロードし、指示欄に共通ルールを記載（例：「必ずナレッジの該当箇所を根拠として示す」）

💡 コツと注意点

  • ナレッジは「関係資料を全部入れる」より、ブランドガイド・要件・FAQ など5〜6点に絞る方が回答精度が安定しやすい
  • ナレッジは自動更新されない。資料改訂のたびに差し替える運用ルールを決めておく
  • Project は PCフォルダを直接読まない。ファイルは手動アップロードが必要（ファイル操作はCoworkの領分）
  • Team/Enterpriseプランでない場合は原則自分専用（他のメンバーとの共有不可）
FEATURE 2

🧩 Artifacts — 会話の結果を「配れる形」にする

一定以上の分量の文書・コード・図・WebアプリをチャットサイドのウィンドウにArtifactとして表示。会話で繰り返し編集し、URLで公開できる。

生成できるコンテンツタイプ

  • 文書（Markdown・構造化テキスト）
  • コード（各種言語）
  • HTML/Reactアプリ（ブラウザ上で動くWebアプリ）
  • SVGグラフィック・Mermaidダイアグラム

作成・公開・カスタマイズ（3ステップ）

  1. チャットで依頼するだけ：「手順書をHTMLのArtifactで作って」と指示すると自動的に右側ウィンドウに生成される
  2. 会話で反復編集：「3章を箇条書きに」「ボタンを青に変えて」など指示のたびにArtifactが更新される。バージョンを切り替えて比較も可能
  3. 公開（Publish）・共有（Share）：Artifactウィンドウ上部の「公開」ボタンでURLを発行。他者の公開ArtifactはCustomizeボタンで自分のコピーを作れる

💡 コツと注意点

  • 短い回答はArtifactにならない。単体で意味を持つ、ある程度の分量が必要。明示的に「Artifactとして作って」と伝えると確実
  • 公開するとコードも見える状態になる。機密情報を含む版を公開しない
  • 他者がCustomizeしても元のArtifactは変更されない（相手側にコピーが作られるだけ）
  • エラーが出たら「Try fixing with Claude」ボタンでエラー内容を渡して修正を試せる
  • Artifactはローカルファイルには書き込まない（ブラウザ上の表示＋ダウンロードのみ）
FEATURE 3

🛠 Cowork — PC作業を「丸ごと任せる」

Claude Codeと同じエージェント基盤をGUI化した機能。「相談」ではなく「作業」を委任し、完成したファイルを受け取る使い方が基本。

主な用途

  • ローカルファイルの直接読み書き（手動アップロード不要）
  • 複数ステップ作業の分解・並列実行（領収書OCR→集計→Excel化など）
  • /schedule による定期タスク（例：毎週金曜に週次報告を自動生成）
  • Coworkプロジェクト：ファイル・指示・メモリをワークスペースに整理（ローカル保存）

前提条件

  • Claude Desktop（最新版）が必須（macOS / Windows）。Web・モバイル単体では動かない
  • 有料プランが必須（Pro / Max / Team / Enterprise）
  • 作業中はアプリを開いたままにする必要あり（PC睡眠で停止）

💡 コツと注意点

  • アクセス権は最小限に。タスクに必要なフォルダだけを渡す。機密ファイルを含むフォルダの付与は避ける
  • 最初は「テスト用フォルダ」で試す。実ファイルを変更できる機能のため、初回はコピーしたデータで挙動を掴む
  • 使用量（トークン消費）が通常チャットより大きい。単純な質問はChat側で行う
  • Coworkのプロジェクト機能はチャットのProjectとは別物（ローカル保存・別管理）
  • 削除操作は許可プロンプトが出る。対象を確認してから承認する
FEATURE 4

⌨ Claude Code — コードはリポジトリごと任せる

ターミナル上でリポジトリを直接読み書きする開発エージェント。コードを「提案する」だけでなく、実際に「手を動かして」ファイルを書き換える。

できること

  • 説明文からの機能実装・バグ修正・リファクタリング・テスト作成
  • コードベース全体の構造把握と質問応答（手動でファイルを渡す必要なし）
  • Git操作の会話化（差分確認・コミット・ブランチ作成・コンフリクト解消）
  • MCP連携でGoogle Drive・Figma・Jira等の外部情報も参照
  • ターミナル以外にWeb版・VS Code/JetBrains拡張・Slack等のインターフェースも提供

インストール方法

# macOS / Linux / WSL（推奨）
curl -fsSL https://claude.ai/install.sh | bash

# Homebrew（macOS / Linux）
brew install --cask claude-code

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

# npm（Node.js 18+ 必要）
npm install -g @anthropic-ai/claude-code

起動方法

  1. cd 対象リポジトリ → claude で起動（初回はブラウザ認証）
  2. まず「このプロジェクトの構成を説明して」でリポジトリを把握させる
  3. その後タスクを自然言語で依頼。/help でコマンド一覧、/resume で過去セッション再開

💡 コツと注意点

  • 必ずGit管理下で使う。実ファイルを書き換えるため、コミット・ブランチでいつでも巻き戻せる状態を先に作る
  • タスクは小さく具体的に。「全部直して」より「このモジュールのこの挙動を直して」が良い
  • テストを「合格条件」として渡す（「テストが通るまで修正して」）と出力品質の検証をエージェント側に組み込める
  • npmインストールが動かない場合は node -v でNode.js 18以上かを確認。満たさないならネイティブインストーラを使う
  • 生成コードの正しさは保証されない。レビュー・テストは人間の責任
